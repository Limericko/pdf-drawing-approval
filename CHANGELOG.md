# 更新日志

## 未发布 - PDM V1 Foundation

- 新增 PDM 零件库基础能力：按“体系文件号 + 管家婆物料号 + 图纸名称 + 版本”解析标准图纸文件名，并兼容文件号后补、物料号缺失待补齐的历史审批数据。
- 新增 PDM 零件、图纸版本、项目使用记录和发布追溯数据模型；同一管家婆物料号全局唯一，同一物料号同一版本不会重复发布。
- 审批通过后自动发布可用的 PDM 图纸版本，并提供零件库、版本历史、项目使用、审批来源链接和元数据修复入口。
- 新增 PDM 历史回填维护服务，可扫描已通过或已打印归档审批，将标准文件名且 PDF 有效的历史图纸补发布到零件库；旧命名、缺文件、无效 PDF 和重复物料版本会跳过并记录原因。

## 0.9.1 - 2026-06-26

- Electron 客户端接入 `electron-updater`，启动后即通过服务端 `/updates/latest.yml` 检查客户端新版，不再依赖用户登录。
- 发现新版后自动下载客户端安装包并显示下载进度，下载完成后弹出安装步骤，用户可打开安装包按向导完成升级。
- 发布流程新增 `latest.yml`、客户端安装包和 blockmap 的运行目录同步，真实服务端 `E:\PDF服务端\pdf-approval\releases\updates` 可直接作为内网更新源。

## 0.9.0 - 2026-06-26

- Electron 客户端新增“打印并归档”：设计师或管理员可在签后 PDF 生成后直接调用 Windows 系统打印。
- 打印前支持设置打印机、份数、页码范围、纸张、方向、彩色/黑白、双面、边距、缩放和打印背景。
- Windows 打印回调成功后自动标记图纸为已打印归档；取消或失败不会归档，浏览器访问仍保留手动归档兜底。
- 更新客户端使用说明、完整用户说明和验证记录，并重新生成客户端、服务端安装包和更新清单。

## 0.8.9 - 2026-06-25

- 修复只升级服务端后，旧客户端打开管理页会显示服务端版本并误判“已是最新”，导致普通用户登录后不提示客户端下载更新的问题。
- 客户端更新检查会优先上报 Electron 已安装外壳版本；旧 Electron 客户端未上报版本时，服务端按未知旧版处理并继续提示新版客户端安装包。
- 管理员“版本更新”面板将当前版本明确标注为“服务端当前版本”，避免和客户端安装版本混淆。
- 重新生成客户端、服务端安装包和更新清单，并同步到真实运行目录 `E:\PDF服务端\pdf-approval\releases`。

## 0.8.8 - 2026-06-25

- 修复窗口尺寸变化时资料页表单、通知偏好和顶部流程向导出现挤压、溢出或文字截断的问题。
- 管理员运维页去掉多余的本机更新日志区块，保留服务端在线更新说明和安装包下载入口。
- 重新生成客户端、服务端安装包和更新清单，并同步到真实运行目录 `E:\PDF服务端\pdf-approval\releases`。

## 0.8.7 - 2026-06-24

- 修复服务端重新安装或升级后 `releases` 更新发布目录被清空，导致 `/updates/latest.json` 返回 `HTTP_404` 的问题。
- 服务端安装器清理旧版本时会保留 `data`、`backups`、`logs`、`releases` 和 `server-config.json`，避免审批数据、日志、备份和更新清单丢失。
- 重新生成客户端、服务端安装包和更新清单，并同步到真实运行目录 `E:\PDF服务端\pdf-approval\releases`。

## 0.8.6 - 2026-06-24

- 业务页面改为路由级按需加载，并在侧边栏悬停或键盘聚焦时预加载目标页面，降低客户端首屏同步加载成本。
- 全量图纸台账关键词搜索增加输入防抖、延迟渲染和刷新提示，减少连续按键造成的重复请求。
- 详情页只在审批状态、文件路径、签名状态等关键字段变化时复查 PDF，PDF 检查失败时提供“重新检查 PDF”入口。
- 长列表、批量历史、操作日志、风险看板和批注项增加 `content-visibility` 渲染优化。
- 服务端新增慢 API 请求日志，可通过 `PDF_APPROVAL_SLOW_REQUEST_MS` 调整阈值；日志不记录请求体、密码或 token。
- 数据库新增操作日志时间线和管理员日志列表索引，提升审批详情追溯和运维日志翻页性能。

## 0.8.5 - 2026-06-23

- 更新清单地址改为由服务端按当前访问地址自动生成，管理端不再要求填写 `update_manifest_url`。
- 服务端窗口新增更新发布信息，可直接查看 `releases` 更新目录并复制 `/updates/latest.json` 清单地址。
- 旧数据库中的 `update_manifest_url` 不再覆盖默认服务端更新源；特殊部署可用服务端环境变量 `PDF_APPROVAL_UPDATE_MANIFEST_URL` 覆盖。

## 0.8.4 - 2026-06-23

- 继续修正服务端安装器自检测逻辑，进程匹配改为明确的 `${PRODUCT_NAME}.exe`。
- 避免 NSIS 模板变量在安装器上下文中匹配到安装包自身，解决从 `releases` 目录运行仍提示无法关闭的问题。

## 0.8.3 - 2026-06-23

- 修复服务端安装包从 `releases\installers\server` 目录运行时，安装器把自身误判为旧服务端进程的问题。
- 安装器现在只检测并关闭真正的应用进程 `PDF图纸审批服务端.exe` / `PDF图纸审批客户端.exe`。

## 0.8.2 - 2026-06-23

- 普通用户登录后会自动检查客户端新版本，发现新版时只提示下载客户端安装包。
- 非管理员更新接口会隐藏服务端安装包链接，避免设计、主管、工艺误下载服务端安装包。
- 打包发布流程会默认同步更新清单和安装包到 `E:\PDF服务端\pdf-approval\releases`。

## 0.8.1 - 2026-06-23

- 服务端内置托管 `updates` 和 `installers` 更新目录，可直接用审批服务端端口发布更新清单和安装包。
- 服务端窗口新增更新发布目录入口，安装包会自动创建 `releases\updates`、`releases\installers\client`、`releases\installers\server` 标准目录。
- 更新清单地址可通过 `http://服务器IP:端口/updates/latest.json` 访问，不再依赖 nginx 或其他静态文件服务。

## 0.8.0 - 2026-06-23

- 新增发布更新清单 `dist/updates/latest.json`，便于在公司内网提供客户端和服务端安装包下载。
