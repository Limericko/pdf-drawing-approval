# Phase 5 WebDAV 受控双向同步实施计划

- 日期：2026-07-14
- 状态：已完成
- 分支：`codex/phase-5-webdav-sync`
- 基线提交：`e3af17b`
- 前置门禁：Phase 0–4 已完成
- 产品方向：统一云端为唯一业务真相，WebDAV 仅作为受控文件交换端

## 目标

在不引入第二套业务数据库、不改变审批/PDM 权威状态的前提下，实现云端与 WebDAV 的受控双向同步：远端 Incoming 文件经过发现、断点下载、隔离校验和哈希去重后进入云端草稿；云端已批准/PDM 已发布版本使用临时名写入、原子移动和回读校验发布到远端；同一路径不同内容进入人工冲突队列，删除默认不传播，所有连接、重试和冲突决策均可审计。

## 固定边界与不变量

- WebDAV 只交换文件，不保存审批状态、问题、PDM current/effective 关系或冲突决策。
- 云端 PostgreSQL、对象存储和 PDM 关系是唯一业务真相；不双写审批状态。
- WebDAV 凭据不进入数据库明文、API 响应或日志。数据库只保存 `credential_ref`；Worker 通过可替换的密钥提供器读取凭据。
- 只允许 `https` 生产端点；开发/测试只允许显式回环 HTTP。禁止 URL 用户信息、查询、片段、重定向到非同源和私网逃逸。
- 入站不直接覆盖已有或已批准版本；同图号/版本不同哈希必须进入冲突。
- 出站只发布 ready 对象；临时名上传成功后才 MOVE 到最终路径，随后回读大小、ETag 和 SHA-256。
- 默认不传播任一端删除。远端缺失记录为 `remote_missing`，云端对象绝不因远端缺失而删除。
- 每个 mapping 的扫描和每个远端路径任务由 PostgreSQL 租约/唯一约束保证单 Worker 所有权。
- 重试有限、指数退避、幂等；永久失败进入 dead job 并可由管理员填写原因重新入队。
- Phase 5 不执行旧 SQLite/文件迁移，不部署正式生产环境；正式密钥服务和切换属于 Phase 6。

## Task 1：验收矩阵、真实 WebDAV 夹具与 RED 门禁

1. 固化 PROPFIND、HEAD、Range GET、PUT、MOVE、DELETE 和 ETag 行为的真实 HTTP WebDAV 测试服务。
2. 为离线、恢复、半途下载、重复发现、重复事件、ETag 变化、MOVE 不支持、回读不一致和远端缺失建立失败测试。
3. 建立连接、映射、同步项、冲突、租约、凭据脱敏和项目隔离矩阵。

退出条件：核心同步流程不依赖源码字符串或内存假对象作为唯一证据。

## Task 2：PostgreSQL schema 与共享契约

新增 migration：

- `webdav_connections`：名称、端点、凭据引用、状态、能力探测、版本和健康时间。
- `webdav_directory_mappings`：项目、连接、Incoming/Published 目录、发布产物类型、扫描间隔、启用状态和租约游标。
- `webdav_sync_items`：方向、远端路径/ETag/大小/时间/哈希、对象/图纸版本引用、状态、尝试和幂等标识。
- `webdav_sync_conflicts`：两端对象/哈希/时间、状态、决策、原因和解决人。

共享 Zod 契约覆盖连接、映射、同步状态、冲突详情/决策、手动扫描、失败重试和诊断汇总。约束必须保证项目边界、远端路径规范化、同 mapping+path 的单一活动项、解决原因和删除不传播。

退出条件：迁移、权限、外键、唯一性、状态机和 restricted-column 集成测试通过。

## Task 3：配置、凭据和 WebDAV 协议客户端

- Worker 配置增加 WebDAV 凭据提供器和网络限制；Web 进程不读取凭据值。
- 本地/测试使用环境注入的引用映射；生产接口预留云密钥服务适配器。
- 协议客户端实现规范化路径编码、同源重定向限制、Basic Auth、PROPFIND 深度、HEAD、Range GET、PUT、MOVE 和受控读取。
- 所有错误分类为可重试离线/限流/5xx、永久认证/权限/协议错误或内容冲突。
- 响应体、单文件大小、目录项数量、超时和重定向次数有硬上限。

退出条件：凭据不出现在错误、日志和响应；协议与 SSRF/路径遍历/重定向安全测试通过。

## Task 4：连接、目录映射与同步管理服务

- 管理员创建/更新/禁用连接和映射，更新使用乐观锁、原因和幂等键。
- 连接测试探测 DAV 能力、认证、目标目录和 MOVE 支持，但不持久化凭据值。
- mapping 必须属于管理员可管理的真实项目；目录不能重叠到产生入站/出站回环。
- 所有变更写不可变审计；API 只返回凭据是否可用和引用，不返回秘密。

退出条件：权限、并发、幂等、目录重叠、凭据缺失和审计测试通过。

## Task 5：入站发现、断点隔离下载与草稿导入

1. Worker 按 mapping 扫描 Incoming，使用 path+ETag+size+mtime 发现变化。
2. 新文件创建 `webdav.inbound` job；下载写入对象存储 staging，并在重试时使用 Range 续传或安全重启。
3. 完成后执行大小、扩展名、MIME、PDF 魔数/解析、文件名/图号版本解析和 SHA-256 去重。
4. 合法文件调用文档模块创建 `webdav_import` 草稿；相同远端事件/相同哈希幂等复用。
5. 同图号版本不同哈希、远端在下载中变化或命中已批准版本时创建冲突，不覆盖。

退出条件：真实服务断点、恢复、伪 PDF、超限、重复事件、并发扫描和冲突测试通过。

## Task 6：出站发布、临时名、原子移动与回读

- PDM 发布成功发出 `pdm.revision.published` Outbox 事件。
- Worker 为所有启用的出站 mapping 创建幂等发布项并选择配置的原始/审查/签后 ready 对象。
- 上传 `${final}.partial-<syncId>`，完成后 MOVE 到最终路径；不支持 MOVE 时仅在能力探测明确允许的等价策略下继续。
- 最终路径已存在且哈希不同则创建冲突；相同哈希视为幂等成功。
- MOVE 后回读 ETag、大小和内容哈希；只有全部一致才标记成功和写审计。

退出条件：重复发布、Worker 中断、临时文件残留、远端预存在、回读不一致和离线恢复测试通过。

## Task 7：冲突、删除不传播、重试和诊断

- 冲突决策：导入为新版本、保留云端并重新发布到重命名路径、仅确认保留远端；每个动作要求原因和最新版本号。
- 所有决策不修改已发布版本内容，不允许静默覆盖。
- 远端缺失只标记 `remote_missing` 并产生可见告警；不发 DELETE，不删除云端对象。
- 同步失败进入统一任务投影和管理诊断；管理员填写原因后重新入队。
- 连接健康、上次成功扫描、积压、冲突、失败和吞吐可查询。

退出条件：删除保护、冲突并发、死信重试、审计脱敏和任务投影测试通过。

## Task 8：同步中心 UI 与领域客户端

- 新增 `syncClient` 和 `/api/v2/webdav-sync` 路由，不引用 legacy `api.ts`、Bearer 或旧 `/api`。
- AppShell 增加管理员“同步中心”；首屏直接显示连接健康、待处理、冲突和最近活动。
- 连接/映射表单使用现有 forms/actions/feedback/data 组件；危险决策必须原因、明确后果和二次确认。
- 冲突详情并列显示云端/远端路径、哈希、大小、修改时间和可选动作。
- 390×844 下保留可访问操作，表格转可扫描列表；键盘、焦点、错误恢复和 reduced motion 通过。

退出条件：五视口 Gallery、axe、溢出、视觉基线和真实浏览器冲突处理通过。

## Task 9：真实闭环与最终门禁

真实闭环至少覆盖：

1. 管理员建立连接和项目目录映射并通过连接测试。
2. WebDAV 新 PDF 被发现、断点下载、校验、去重并进入图纸草稿。
3. 远端重复事件不创建第二份草稿。
4. 云端审批/PDM 发布触发临时上传、MOVE 和回读校验。
5. 同一路径不同内容进入冲突，管理员人工决策且审计可查。
6. 远端删除只产生缺失告警，云端对象/版本保持不变。
7. WebDAV 离线形成有限重试，恢复后自动追赶且无重复。

最终命令至少包含：

```powershell
npm run e2e:ui

```

最终扫描：

- 平台同步 import graph 对 legacy `api.ts`、Bearer 和非 `/api/v2` 路径为 0。
- 凭据值、Authorization、Cookie、PDF 内容不进入日志、审计、Problem Details 或客户端 bundle。
- 新 UI 硬编码颜色、数字 z-index 和调试输出为 0。
- PostgreSQL 约束和真实 WebDAV HTTP 行为共同证明无静默覆盖、删除不传播和幂等恢复。

## 完成判定

只有 Task 1–9 均有当前代码、真实 PostgreSQL/对象存储/WebDAV 服务、Worker 和浏览器证据，且离线恢复、重复事件、冲突和删除保护全部通过，Phase 5 才可标记完成。
