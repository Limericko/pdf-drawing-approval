# PDF 图纸审批系统第四版方案

日期：2026-06-17

## 版本定位

第四版目标是把第三版从“可试运行的签审闭环系统”升级为“更适合长期现场使用的产品化系统”。

本版本主线已确认：

1. 签名框模板。
2. 批量上传。
3. 批量签后 PDF 处理。
4. 运维风险看板。
5. 轻量版本追溯预留。

第四版继续保持局域网内部使用、Windows 优先、坚果云本地同步目录为文件底座，不引入坚果云 API、云存储、复杂审批流引擎或 CA 数字签名。

## V3 基线

第三版已经具备：

- 网页上传单张 PDF。
- PDF 预览上放置设计、主管、工艺三个签名框。
- 固定主管和固定工艺并行审核。
- 双审通过后自动生成签后 PDF。
- 设计师或管理员打印归档。
- 评论、问题、操作日志、CSV 追溯、系统诊断、备份。
- 管理员删除图纸时清理受管文件、历史签后文件和空项目文件夹。

第四版不推翻这些能力，只围绕“减少重复操作”和“提升现场运维可见性”增量扩展。

## 已确认决策

- V4 主线按“签名框模板 + 批量上传”推进。
- 固定主管 + 固定工艺并行审核继续保留。
- 批量上传仍写入坚果云标准目录 `02-审批中\项目名\`。
- 签名框模板先支持全局模板和项目模板，不做复杂自动匹配规则。
- 批量上传中每张图纸都有独立签名框位置；模板只能作为初始化或批量套用，不作为整批最终位置。
- 批量上传中单个文件失败不阻塞其它文件。
- 批量处理不做一键调用打印机，仍由设计师或管理员人工确认打印。
- 版本追溯先做轻量提示和同零件版本列表，不做完整版本树。

## 设计原则

- 先降低重复劳动，再扩展复杂流程。
- 批量操作必须逐项可追溯，不能只有一个模糊成功或失败。
- 涉及文件写入、签名生成、归档、删除等动作时，数据库记录和磁盘文件状态必须一致。
- 所有批量能力都应允许部分成功，并给出逐项错误原因。
- 不为了少量用户引入复杂配置；5-10 人团队优先选择可理解、可维护的规则。

## 目标用户与角色

### 设计师

- 维护或使用签名框模板。
- 一次上传多张 PDF。
- 为批量图纸统一项目名。
- 批量套用签名框模板后，逐张检查和微调各自签名框。
- 查看同零件历史版本提示。
- 对待打印图纸批量检查签名状态。

### 主管 / 工艺

- 继续使用待我审核。
- 可查看图纸详情中的同零件历史版本提示。
- 不参与签名框模板维护。

### 管理员

- 管理签名框模板。
- 查看批量提交记录。
- 批量重新生成签后 PDF。
- 查看运维风险看板并跳转处理。
- 导出更完整的追溯报表。

## 核心模块

### 1. 签名框模板

签名框模板用于复用常见图框签字栏位置。

模板内容：

- 模板名称。
- 适用项目名，可为空表示通用模板。
- 适用页码。
- 设计、主管、工艺三个签名框位置。
- 创建人和更新时间。

位置继续使用第三版归一化字段：

- `pageNumber`
- `xRatio`
- `yRatio`
- `widthRatio`
- `heightRatio`

模板来源：

1. 从当前审批单已保存的签名框另存为模板。
2. 在系统管理中手工新建或编辑模板。

模板使用：

1. 提交图纸时选择模板。
2. 系统把模板中的三个签名框复制到当前图纸。
3. 用户仍可在 PDF 预览上微调。
4. 提交后按审批单自己的 `signature_placements` 保存，不与模板继续绑定。

边界：

- 套用模板不会覆盖已手动编辑的位置，除非用户明确点击“套用模板”。
- 删除模板不影响已经提交的审批单。
- 第一版模板不自动判断图纸尺寸是否完全匹配，只在界面提示模板来源和页码。

### 2. 批量上传

提交图纸页升级为支持单张和多张。

批量上传流程：

1. 设计师选择多个 PDF。
2. 前端展示文件列表。
3. 每个文件独立校验扩展名、PDF 文件头、文件名格式。
4. 文件名符合 `零件名-a0A0.pdf` 时自动解析零件名和版本。
5. 用户填写统一项目名，也可逐项修正零件名和版本。
6. 用户可选择一个模板批量初始化签名框，也可为单个文件选择不同模板。
7. 系统为每个文件维护独立签名框位置。
8. 用户逐张打开预览，检查或调整该文件自己的签名框。
9. 提交前系统逐项检查三类签名框是否齐全。
10. 提交时逐个创建审批单。
11. 每个文件写入 `02-审批中\项目名\`。
12. 页面显示成功数、失败数和逐项结果。

签名框位置规则：

- 每个批量上传项都有自己的 `placements`。
- 对整批套用模板时，只把模板位置复制到每个文件的 `placements` 初始值。
- 单个文件调整后，不影响同批其它文件。
- 文件列表显示每个文件的签名框状态：未放置、已套用模板、已人工调整。
- 提交时，后端按文件逐项校验设计、主管、工艺三个签名框，不能用批次级统一位置替代单项位置。

异常规则：

- 非 PDF 或 PDF 头无效：该项失败。
- 文件名无法解析：该项保留在列表中等待用户修正。
- 签名框不完整：该项失败或阻止提交，提示先补齐该文件的设计、主管、工艺签名框。
- 项目、零件、版本重复：该项失败并提示已有审批单。
- 某一项写入失败：不回滚其它成功项。
- 全部失败时不创建批量提交成功状态。

### 3. 批量签后 PDF 处理

“全部图纸”页面增强批量操作。

支持范围：

- 仅设计师和管理员可操作。
- 仅对 `approved_for_print` 状态记录开放批量签后处理。
- 签名未生成或失败的记录可批量重新生成。
- 批量打印归档前必须检查签后 PDF 已生成。

批量重新生成签后 PDF：

1. 用户勾选多张已通过待打印图纸。
2. 点击“批量重新生成签后 PDF”。
3. 后端逐张调用现有签名生成流程。
4. 返回每张图纸的成功或失败原因。
5. 前端刷新列表和提示结果。

批量归档：

1. 用户勾选多张待打印图纸。
2. 系统检查每张图纸是否有签后 PDF。
3. 不满足条件的记录列出原因。
4. 满足条件的记录可批量标记打印归档。

边界：

- 不自动打开打印窗口。
- 不合并 PDF。
- 不覆盖已有签后 PDF，仍按现有规则生成新文件。

### 4. 运维风险看板

系统管理页增加“风险看板”区域，用于把影响日常使用的风险集中展示。

风险项：

- 审批根目录未配置。
- 标准目录缺失。
- 标准目录不可写。
- 最近扫描失败。
- 最近备份过期。
- 签名失败待处理。
- 文件丢失待处理。
- PDF 无效待处理。
- 关键角色未配置签名。

每个风险项包含：

- 风险级别：正常、提醒、异常。
- 数量或状态。
- 简短说明。
- 处理入口。

示例入口：

- 文件丢失 -> `#/approvals?status=file_missing`
- 签名失败 -> `#/approvals?signatureStatus=failed`
- 备份过期 -> 系统管理备份面板
- 签名未配置 -> 签名配置状态

### 5. 轻量版本追溯

V4 不做完整版本树，只做同项目、同零件名的版本提示。

上传时：

- 如果系统发现同项目、同零件名已有其它版本，显示提示。
- 提示内容包括版本、状态、提交时间。

详情页：

- 显示“同零件其它版本”列表。
- 列表包含版本、状态、提交时间、归档时间和打开入口。

报表：

- CSV 增加同零件版本数。
- 可选增加上一版本状态。

## 数据设计

### `signature_templates`

新增签名框模板表。

```sql
CREATE TABLE IF NOT EXISTS signature_templates (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  project_name TEXT,
  placements_json TEXT NOT NULL,
  created_by_user_id INTEGER,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  updated_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (created_by_user_id) REFERENCES users(id)
);
```

`placements_json` 保存三个角色的位置数组，结构与 `signature_placements` API 保持一致。

### `batch_submissions`

记录一次批量提交。

```sql
CREATE TABLE IF NOT EXISTS batch_submissions (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  created_by_user_id INTEGER,
  project_name TEXT NOT NULL,
  status TEXT NOT NULL CHECK (status IN ('running', 'completed', 'failed', 'partial')),
  total_count INTEGER NOT NULL,
  success_count INTEGER NOT NULL DEFAULT 0,
  failed_count INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  finished_at TEXT,
  FOREIGN KEY (created_by_user_id) REFERENCES users(id)
);
```

### `batch_submission_items`

记录批量提交中的每个文件。

```sql
CREATE TABLE IF NOT EXISTS batch_submission_items (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  batch_id INTEGER NOT NULL,
  file_name TEXT NOT NULL,
  approval_id INTEGER,
  status TEXT NOT NULL CHECK (status IN ('pending', 'completed', 'failed')),
  error_message TEXT,
  placement_state TEXT,
  created_at TEXT NOT NULL DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (batch_id) REFERENCES batch_submissions(id),
  FOREIGN KEY (approval_id) REFERENCES approvals(id)
);
```

`placement_state` 可记录提交时该项签名框来源，例如 `manual`、`template`、`missing`。审批成功后，实际位置仍写入 `signature_placements`，保证每张图纸独立追溯。

## 接口设计

### 签名框模板

```text
GET    /api/signature-templates
POST   /api/signature-templates
PUT    /api/signature-templates/:id
DELETE /api/signature-templates/:id
POST   /api/approvals/:id/signature-templates
```

权限：

- 设计师和管理员可读取模板。
- 设计师和管理员可创建模板。
- 管理员可编辑和删除所有模板。
- 设计师只能编辑或删除自己创建的模板。

### 批量上传

```text
POST /api/submissions/batch-upload
POST /api/submissions/batch
GET  /api/submissions/batches
GET  /api/submissions/batches/:id
```

`batch-upload` 接收多个 PDF 暂存文件并返回逐项解析结果。

`batch` 确认提交并逐项创建审批单。

确认提交时，每个 item 必须携带自己的签名框位置：

```json
{
  "projectName": "LS-300N",
  "items": [
    {
      "uploadId": "tmp-1",
      "partName": "前面板",
      "version": "a0A0",
      "placements": [
        { "role": "designer", "pageNumber": 1, "xRatio": 0.6, "yRatio": 0.82, "widthRatio": 0.08, "heightRatio": 0.04 }
      ]
    }
  ]
}
```

### 批量审批后处理

```text
POST /api/approvals/batch/generate-signed-pdf
POST /api/approvals/batch/mark-printed
```

返回逐项结果：

```json
{
  "total": 3,
  "success": 2,
  "failed": 1,
  "items": [
    { "approvalId": 1, "status": "completed" },
    { "approvalId": 2, "status": "failed", "error": "SIGNATURE_ASSET_MISSING" }
  ]
}
```

### 风险看板

```text
GET /api/system/risks
```

返回风险项列表。

## 前端设计

### 提交图纸页

页面结构：

- 上传区域支持多选。
- 文件列表显示每个文件的解析状态。
- 顶部提供统一项目名和“批量套用模板”。
- 每个文件行显示签名框状态。
- 单个文件可选择不同模板、展开预览并微调自己的签名框。
- 底部显示批量提交按钮和结果摘要。

交互重点：

- 单张上传流程仍保持简单。
- 多张上传时默认只展开第一张预览，避免页面过重。
- 切换文件预览时，保存当前文件的签名框状态，再加载目标文件自己的签名框状态。
- 如果某张图纸的签名位置与模板不一致，用户只调整该图纸，不影响整批其它图纸。
- 失败项保留在列表中，便于修正后再次提交。

### 全部图纸页

增强批量操作栏：

- 已选数量。
- 删除所选。
- 批量重新生成签后 PDF。
- 批量标记打印归档。

批量操作结果使用列表弹窗展示，避免只弹一个笼统提示。

### 系统管理页

新增风险看板区域：

- 放在运维追溯页顶部。
- 风险项使用紧凑列表或状态卡。
- 每项提供“去处理”按钮。

新增签名模板管理：

- 可放在系统管理独立标签或提交页模板选择旁。
- V4.1 先做简单表格：名称、适用项目、创建人、更新时间、操作。

## 错误处理

- 批量接口不因单项失败直接返回 500，除非整个请求无法解析或系统级错误。
- 单项业务失败用 `items[].status = failed` 和 `items[].error` 表达。
- 文件写入失败、重复审批单、签名缺失、签名框缺失都必须可读。
- 高风险批量操作写入 `operation_logs`。

## 验收标准

V4.1：

- 从审批详情页保存模板。
- 提交新图纸时套用模板。
- 套用后可微调并保存。
- 签后 PDF 使用新位置生成。

V4.2：

- 同批 3 个有效 PDF 创建 3 张审批单。
- 同批 1 个无效 PDF 不影响其它有效文件。
- 同批不同图纸可保存不同签名框位置。
- 某一项缺少签名框时，只阻止或失败该项，不影响其它位置完整的文件。
- 重复版本给出逐项错误。
- 文件写入坚果云标准目录。

V4.3：

- 多张待打印图纸可批量重新生成签后 PDF。
- 签名失败逐项显示失败原因。
- 批量归档只处理签后 PDF 已生成的记录。

V4.4：

- 管理端显示风险看板。
- 风险项能跳转到处理入口。
- 备份过期、目录异常、签名失败、文件丢失能被识别。

## 不纳入 V4

- CA/证书数字签名。
- 多级审批和动态审批人路由。
- 坚果云 API 集成。
- 自动调用 Windows 打印机。
- 完整版本树、变更单和 BOM 关联。
