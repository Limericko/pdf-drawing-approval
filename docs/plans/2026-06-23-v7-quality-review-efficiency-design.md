# PDF 图纸审批系统 V7 上线质量与审图效率增强方案

日期：2026-06-23

## 背景

系统已经具备局域网审批主流程：网页/客户端提交 PDF、固定主管和工艺并行审核、签名框定位、自动生成签后 PDF、图纸批注、审查版 PDF、批量上传、批量签后处理、用户资料、通知偏好、服务端安装包和客户端安装包。

当前不再优先扩展复杂审批流。V7 聚焦真实上线后的稳定使用：同事能快速连上服务端，管理员能发现和恢复问题，审核人能更快看图和批注，系统在图纸量增长后仍保持可用。

## 目标

- 增强客户端和服务端连接诊断，减少“服务端正常但同事连不上”的排障成本。
- 补齐客户端/服务端版本信息和兼容性提示，避免只升级一端导致不可预期错误。
- 优化 PDF 审图效率：页码跳转、缩略页导航、适高/适宽/100% 快捷切换、批注筛选和连续标注。
- 增加自动运维能力：定时备份、定时清理、备份可用性校验。
- 为审批列表、版本追溯和风险看板补数据库索引，支撑更多图纸。
- 拆分过大的核心页面，降低后续继续迭代时的改动风险。

## 非目标

- 不做项目级动态审核人配置，继续保持固定主管和固定工艺并行审核。
- 不接入坚果云 API，继续使用服务器本地同步目录。
- 不做 CA/证书数字签名。
- 不接入企业微信、飞书或短信通知。
- 不重写 PDF 渲染为完整 PDF.js Viewer。
- 不把 Tauri 托盘助手作为正式客户端继续推进；它保留为历史实验或后续清理对象。

## 推荐方案

V7 拆成五个可独立验证的批次。

### 批次 A：连接诊断与版本兼容

后端扩展 `/health` 或新增 `/api/system/version`，返回服务端版本、构建时间、API 兼容版本、局域网地址、端口和运行模式。客户端登录页和 Electron 连接页增加“连接自检”，一次性检查：

- 服务端是否可达。
- `/health` 是否返回正常。
- API 版本是否兼容当前客户端。
- PDF 文件访问基准路径是否可用。
- 用户输入的是 `127.0.0.1`、`localhost` 还是真实局域网地址。

服务端窗口继续显示真实局域网 IP，并增加“复制客户端地址”能力。客户端无法连接时，提示检查服务端是否运行、防火墙是否放行端口、服务器地址是否为局域网 IP。

### 批次 B：PDF 审图效率二期

在现有 `PdfViewportControls` 基础上增强查看能力：

- 页码输入和上一页/下一页。
- 缩略页导航，点击跳转到指定页面。
- 适宽、适高、100%、放大、缩小、拖动浏览保持一致。
- 批注列表支持未处理、已处理、我创建的、类型筛选。
- 连续标注模式：保存一个批注后保持当前工具和颜色，适合连续标多个问题。
- 审查版 PDF 生成前展示当前批注汇总数量。

所有增强都继续使用比例坐标，不改变 `approval_annotations` 和 `signature_placements` 的坐标语义。

### 批次 C：自动运维与备份校验

在服务端增加轻量计划任务配置，不引入外部调度系统：

- 管理员可启用每日自动备份，配置执行时间。
- 管理员可启用每日自动清理，复用现有清理策略。
- 增加“校验备份”动作：检查备份目录是否包含 SQLite 主库和可选 WAL/SHM 文件，能否读取基本元数据。
- 风险看板显示最近一次自动备份、最近一次自动清理和最近一次备份校验结果。

自动任务失败只写操作日志和风险看板，不中断审批服务。

### 批次 D：数据库索引与长期使用性能

保留当前 SQLite 和 `node:sqlite`。新增幂等索引：

- `approvals(status, submitted_at, id)`：全部图纸状态筛选和风险入口。
- `approvals(signature_status, submitted_at, id)`：签名失败和待签名筛选。
- `approvals(project_name, part_name, submitted_at, id)`：同零件版本追溯。
- `approvals(current_file_path)`：文件监听和重绑定查找。
- `approvals(submitted_by_user_id, submitted_at, id)`：后续个人提交记录。

如果后续关键字搜索量明显增加，再考虑 FTS5；V7 先不引入全文检索表。

### 批次 E：核心页面拆分与历史方案收口

不改变视觉和业务行为，先做低风险拆分：

- `ApprovalDetailPage.tsx` 拆出批注面板、签名面板、浮动协同面板、异常处理面板。
- `SettingsPage.tsx` 拆出目录与通知、用户管理、签名模板、运维追溯、服务日志。
- `api.ts` 后续可按领域拆分，但 V7 第一轮只在新增接口时建立 `api/system.ts` 或小型 helper，避免一次性大迁移。
- Tauri 托盘助手文档标注为历史实验，不再出现在主部署路径的推荐步骤中。

## 数据与配置

新增设置项建议继续走 `settings` 表：

- `maintenance_auto_backup_enabled`
- `maintenance_auto_backup_time`
- `maintenance_auto_cleanup_enabled`
- `maintenance_auto_cleanup_time`
- `client_api_compat_version`

新增记录表可选：

- `maintenance_runs`
  - `id`
  - `kind`: `auto_backup`、`auto_cleanup`、`backup_validation`
  - `status`: `running`、`completed`、`failed`
  - `message`
  - `metadata_json`
  - `started_at`
  - `finished_at`

如果实现时发现 `backup_runs` 和 `operation_logs` 已能覆盖自动备份，`maintenance_runs` 可只记录自动清理和备份校验，避免重复数据。

## 权限

- 连接自检：未登录可访问基础健康检查，不暴露 SMTP、数据库路径、签名文件路径等敏感信息。
- 详细运维诊断：仅管理员可访问。
- 自动备份、自动清理、备份校验：仅管理员可配置和手动触发。
- PDF 审图增强：查看控制所有可进入详情页的用户可用；批注筛选所有可看批注的用户可用；连续标注仅主管、工艺、管理员可用。

## 风险

- 自动备份和自动清理需要避免和人工操作同时执行，第一版使用进程内互斥锁即可。
- 服务端不是 Windows Service，隐藏到托盘后仍依赖用户会话；文档要继续说明“退出服务端”才是真正停止。
- PDF 缩略页会增加渲染成本，需要懒渲染或低分辨率渲染，不能一次性渲染所有高清页面。
- 大页面拆分必须保持行为不变，先覆盖源级测试和现有回归，再做视觉调整。

## 验收标准

- 客户端连接页能显示服务端可达、版本兼容、地址建议和失败原因。
- 服务端窗口能复制真实局域网访问地址。
- PDF 预览支持页码跳转、上一页/下一页、缩略页导航、适高和连续标注。
- 批注列表能按处理状态、作者和类型筛选。
- 管理员可启用自动备份、自动清理，并能查看最近执行结果。
- 管理员可校验某个备份目录是否可读。
- 数据库迁移后新增索引存在，旧数据不丢失。
- `npm test` 和 `npm run build` 通过。
- 客户端和服务端安装包仍可重新打包。
