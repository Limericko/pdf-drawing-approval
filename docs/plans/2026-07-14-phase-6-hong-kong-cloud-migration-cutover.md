# Phase 6：中国香港云部署、生产密钥、迁移与切换实施计划

## 目标与固定边界

Phase 6 将已经完成的 v2 平台部署到阿里云中国香港，并把旧 SQLite、文件和必要配置迁移为唯一云端业务真相。办公室和家庭用户使用同一 HTTPS 域名直接登录，不依赖 VPN，不保留办公室本地业务服务器，也不建设本地与云端的业务双写。

固定容量与可用性目标：30–100 个账号、10–30 人并发；两个不同可用区的无状态应用实例；托管 PostgreSQL 高可用；私有 OSS；月度 SLO 99.9%；RPO 不超过 5 分钟；RTO 不超过 30 分钟；正式切换窗口 1–2 小时。

正式上线前仍需确定的外部输入：

- 正式域名及 DNS 管理权限。
- 阿里云账号、`cn-hongkong` 区域资源创建权限、两个可用区及费用确认。
- 正式 SMTP 和 WebDAV 端点、账号及允许主机名。
- 切换日期、公告对象和最终 Go/No-Go 负责人。

这些输入缺失时可以完成代码、镜像、基础设施模板和脱敏迁移演练，但不能声称生产部署或正式切换完成。

## Task 1：生产配置与密钥文件注入

1. 所有数据库 URL、OSS 密钥、SMTP 密码、身份 keyring 和 WebDAV 凭据支持只读绝对路径 `*_FILE` 注入。
2. 同一字段的直接值与文件值同时存在时启动失败；文件不是普通文件、权限过宽、包含 NUL/空值或超限时启动失败。
3. 生产 Compose 只引用 `/run/secrets` 的 tmpfs/只读文件，不把密钥写入镜像、Compose、Terraform 变量、日志或 Terraform state。
4. 阿里云 KMS Secrets Manager 仅创建秘密容器和 RAM 最小权限；秘密版本由受控上线步骤写入，运行主机在启动时获取并原子生成 `/run/secrets/pdf-approval/*.secret`。
5. keyring 支持保留旧版本和切换 `currentVersion`，轮换时先扩展、再切换、最后在观察期后收缩。

退出条件：配置单元测试、生产拒绝测试、脱敏扫描和容器内只读秘密挂载测试通过。

## Task 2：单一签名镜像与生产 Compose

1. 多阶段 Dockerfile 构建 React 产物并安装最小运行依赖；以固定非 root UID 运行。
2. 同一镜像提供 `web`、`worker`、`migration`、`bootstrap-admin` 四个入口；镜像以 digest 部署。
3. Web 和 Worker 根文件系统只读；临时目录与 WebDAV staging 使用受限 tmpfs；业务文件只进入 OSS。
4. Web 暴露内部 HTTP 端口，仅接受负载均衡安全组访问；Worker 不开放端口。
5. 健康检查区分 liveness/readiness；数据库、OSS 和 schema 未就绪时不得加入负载均衡。
6. 容器设置停止宽限期、资源上限、日志轮换、`no-new-privileges`、capability drop 和重启策略。

退出条件：镜像可重复构建、SBOM/依赖扫描无高危未解释项、容器以非 root 和只读根文件系统通过健康及业务冒烟。

## Task 3：阿里云香港可重复基础设施

使用 Terraform 管理：

- 一个 VPC、两个可用区的私有 vSwitch、受限安全组和受控出站。
- 两台 ECS 应用主机，分布在不同可用区；实例 RAM Role 只允许拉取指定 ACR 镜像、读取指定秘密和访问指定 OSS Bucket。
- 公网 ALB、HTTPS 证书、健康检查和两个实例的 server group；WAF 位于 ALB 前。
- RDS PostgreSQL 高可用实例，私网访问、TLS、自动备份、日志备份和 5 分钟内恢复点目标。
- 私有 OSS Bucket，版本化、KMS 服务端加密、阻止公共访问、生命周期和跨故障域备份。
- KMS CMK、Secrets Manager 秘密容器、日志服务、监控告警和预算告警。

Terraform 不创建秘密版本，不接收数据库密码、OSS Secret、SMTP 密码或应用 keyring 明文。`plan` 必须可审阅；`apply` 需要人工确认费用和目标账号/区域。

退出条件：`fmt`、`validate`、静态安全扫描、空白账号 plan 审阅和资源策略测试通过。

## Task 4：发布、排空与一键回退

1. 发布前验证镜像签名/digest、数据库迁移兼容性、备份新鲜度和当前实例健康。
2. 先从 ALB 摘除实例 A，等待连接排空，更新并验证；再对实例 B 重复。
3. 任一实例 readiness 失败立即停止，不替换下一台。
4. 回退只切回上一镜像 digest；数据库迁移保持 expand/contract，不自动执行破坏性回滚。
5. 发布、回退、操作者、镜像 digest、迁移版本和结果进入不可变部署记录。

退出条件：本地双实例等价演练和云端单实例故障/滚动发布演练均不造成整体不可用。

## Task 5：SQLite 与文件迁移工具

迁移分为 `inventory`、`preflight`、`import`、`verify` 和 `delta` 五个显式阶段，重复执行必须幂等：

1. 冻结旧库副本，通过 SQLite 在线备份或停写后的数据库/WAL 一致副本读取，不直接修改旧库。
2. 扫描逐表计数、未知状态、孤儿引用、重复业务键、用户角色、路径越界、缺失文件和非 PDF/损坏 PDF。
3. 保留可安全复用的稳定 ID；其他记录写入确定性的 legacy-to-v2 ID 映射表和报告。
4. 现有账号迁移为待邀请/禁用账号，不沿用旧密码作为公网认证；主管和工艺继续并行双审。
5. 文件先计算 SHA-256，再上传临时对象，HEAD/GET 回读验证大小和哈希，最后在同一数据库事务中写 `storage_objects` 和业务引用。
6. 原始 PDF、审阅标注 PDF、签后 PDF、签名 PNG 和 PDM 版本分别映射；缺失、重复、无效和无法映射项进入异常报告，不静默跳过。
7. `verify` 校验逐表计数、关键聚合、外键、唯一约束、当前版本指针、对象存在/大小/哈希和 PDF 可解析性。
8. `delta` 只处理演练基线之后的新增或变化记录，并生成最终 Go/No-Go 报告。

迁移报告使用稳定 JSON schema，包含源指纹、运行 ID、工具/代码版本、开始结束时间、计数、异常、对象哈希、耗时和是否允许切换；报告本身不得包含密码、会话 token、签名图像或 PDF 内容。

退出条件：脱敏完整副本至少两次演练结果一致；第二次全量执行无重复写；增量迁移在正式窗口预算内完成；所有差异有解释。

## Task 6：备份、恢复、监控和安全门禁

- PostgreSQL：自动备份、日志备份、PITR 和独立恢复实例演练。
- OSS：版本化、删除保护、对象清单和抽样/全量哈希复核。
- 应用：只备份非秘密配置版本、镜像 digest、迁移报告和基础设施 state；秘密由 KMS 管理并单独验证可恢复性。
- 监控：ALB 5xx/延迟、实例 readiness、RDS 连接/存储/复制、OSS 错误、Worker 心跳、任务积压/死信、WebDAV 冲突和 SMTP 健康。
- 安全：WAF、TLS、安全头、MFA、邀请制、项目越权、CSRF、限流、恶意/伪 PDF、依赖/镜像/秘密扫描，高危问题必须为零。

退出条件：单实例故障不影响服务；数据库和对象恢复演练证明 RPO ≤ 5 分钟、RTO ≤ 30 分钟；告警可到达指定负责人。

## Task 7：正式切换

1. T-7 天完成账号邀请、MFA 绑定、演练报告和用户公告；T-1 天确认备份、域名 TTL、资源健康和回滚负责人。
2. 维护窗口开始后旧系统写入冻结，WebDAV 入站暂停，创建 SQLite、WAL、文件和配置最终备份。
3. 执行增量迁移与全量 verify；管理员、设计师、主管、工艺分别完成登录、任务、PDF、问题、并行双审、签章、PDM、归档和 WebDAV 冒烟。
4. Go/No-Go 负责人签字后切换 DNS/WAF/ALB，恢复 WebDAV 并开放新系统写入。
5. 观察期内旧系统只读；监控错误、延迟、任务积压、同步冲突和对象一致性。
6. 若 Go/No-Go 前失败，恢复旧域名和旧系统；若新系统开放写入后必须回退，先冻结并导出云端增量，禁止用旧库覆盖新数据。

## 最终完成门禁

Phase 6 只有在以下证据全部存在时才可标记完成：

- 阿里云香港两个可用区的真实资源、HTTPS 域名、WAF/ALB、RDS、OSS、KMS 和监控证据。
- 生产秘密不在代码、镜像、Terraform state、日志和客户端 bundle 中，并完成一次轮换演练。
- 正式迁移报告证明数据库、对象、PDF、角色和关键聚合无未解释丢失。
- 真实滚动发布、单实例故障、备份恢复和角色浏览器冒烟通过。
- SLO、RPO、RTO、安全、性能和回滚门禁达到总规格。
