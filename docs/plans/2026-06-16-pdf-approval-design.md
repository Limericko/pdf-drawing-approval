# PDF 图纸审批系统第一版设计方案

## 目标

为 5-10 人机械设计团队建立一个局域网内使用的 PDF 图纸在线审批系统，减少纸质预审浪费，同时保留现有坚果云文件管理习惯。

第一版目标不是替代坚果云，而是在坚果云同步目录之上增加自动识别、审批流转、意见记录和可打印状态管理。

## 使用环境

- 团队电脑均为 Windows。
- 系统只在公司局域网内使用。
- 一台办公室 Windows 电脑作为审批服务器，保持开机。
- 审批服务器安装坚果云客户端，并同步指定图纸审批目录。
- 其他成员通过浏览器访问审批系统。

## 推荐架构

```text
设计师提交 PDF 到坚果云同步目录
        ↓
审批服务器监听本地同步目录
        ↓
识别文件名和项目目录
        ↓
生成审批单并通知主管、工艺
        ↓
主管和工艺并行在线审核
        ↓
通过后进入待打印，驳回后返回修改
```

## 目录结构

建议在坚果云中建立统一根目录：

```text
图纸审批/
  01-待提交/
    项目A/
  02-审批中/
    项目A/
  03-已驳回/
    项目A/
  04-已通过待打印/
    项目A/
  05-已打印归档/
    项目A/
```

设计师只需要把 PDF 放入：

```text
图纸审批/01-待提交/项目名/零件名-a0A0.pdf
```

其他状态目录由系统自动维护。

## 文件命名规则

第一版强制使用：

```text
零件名-a数字A数字.pdf
```

示例：

```text
轴承座-a0A0.pdf
轴承座-a1A0.pdf
上盖板-a0A1.pdf
```

系统解析规则：

- 项目：来自 `01-待提交` 下一级文件夹名称。
- 零件名：文件名中版本号前面的部分。
- 版本：`a数字A数字`。
- 小版本：`a数字`。
- 大版本：`A数字`。

不符合命名规则的文件不进入审批，进入异常列表，由设计师改名后重新提交。

## 审批角色

第一版使用固定人员：

- 设计师：提交 PDF，查看状态和意见。
- 主管：固定审核人。
- 工艺：固定审核人。
- 打印人员：查看已通过待打印图纸，完成打印后归档。
- 管理员：配置目录、账号、邮件参数。

## 审批流程

采用主管和工艺并行审核：

```text
设计师提交 PDF
  ↓
系统生成审批单
  ↓
同时通知主管和工艺
  ↓
主管审核：通过 / 驳回 / 填写意见
工艺审核：通过 / 驳回 / 填写意见
  ↓
两人都通过：可打印
任意一人驳回：需修改
```

驳回后不覆盖原文件。设计师按意见导出新 PDF，并提交新版本：

```text
轴承座-a0A0.pdf  已驳回
轴承座-a1A0.pdf  重新提交
```

系统按“项目 + 零件名”聚合历史版本。

## 状态设计

单个审核人的状态：

- 待审
- 通过
- 驳回

审批单总状态：

- 审批中
- 已驳回
- 已通过待打印
- 已打印归档
- 文件名异常

状态规则：

- 主管和工艺均为通过时，总状态为已通过待打印。
- 任意一方驳回时，总状态为已驳回。
- 审批中允许另一方继续查看，但不再改变已驳回结论，除非管理员重新打开审批。
- 已通过待打印后，由打印人员或管理员标记为已打印归档。

## 页面范围

### 登录页

使用简单账号密码登录。第一版不接企业身份系统。

### 待我审核

主管或工艺登录后看到自己的待办：

- 项目
- 零件名
- 版本
- 提交时间
- 当前状态
- 打开审批

### 全部图纸

用于查询审批进度：

- 按项目筛选
- 按零件名搜索
- 按状态筛选
- 查看主管和工艺审核状态

### 图纸详情

核心审批页面：

- PDF 在线预览
- 项目、零件名、版本
- 提交时间和文件路径
- 主管审核状态、意见和时间
- 工艺审核状态、意见和时间
- 历史版本列表
- 通过按钮
- 驳回按钮和意见输入

### 配置页

管理员使用：

- 坚果云本地根目录
- 主管账号和邮箱
- 工艺账号和邮箱
- SMTP 邮件配置
- 是否自动移动文件
- 系统访问地址提示

## 通知方案

第一版采用：

- 网页待办
- 邮件通知
- 浏览器通知

新审批单生成后：

1. 主管和工艺的待办列表出现任务。
2. 系统发送邮件通知。
3. 如果审批网页已打开并授权通知，弹出浏览器提醒。

邮件内容包含：

- 项目
- 零件名
- 版本
- 提交时间
- 审批链接

真正的 Windows 系统桌面弹窗放到第二阶段，通过托盘小助手实现。

## 文件处理规则

- 监听 `01-待提交` 目录下新增 PDF。
- 检测到文件后等待文件稳定，避免坚果云尚未同步完成时处理半文件。
- 文件稳定后解析项目、零件名和版本。
- 命名合法时创建审批单，并将文件移动到 `02-审批中`。
- 命名非法时记录异常，不移动或移动到异常区域，具体实现阶段再定。
- 审批通过后移动到 `04-已通过待打印`。
- 审批驳回后移动到 `03-已驳回`。
- 标记已打印后移动到 `05-已打印归档`。

## 第一版不做

- 电子签章
- 手写批注
- 复杂 PDF 标注
- 手机端深度适配
- 外网访问
- 多项目动态审核人
- 完整图纸管理系统
- 替代坚果云存储

## 风险与约束

- 审批服务器电脑需要保持开机。
- 坚果云同步可能存在延迟，必须做文件稳定检测。
- 文件名规则需要团队统一执行。
- 局域网访问需要固定服务器 IP 或局域网主机名。
- 邮件通知依赖 SMTP 配置和邮箱授权码。

## 后续增强

- Windows 托盘提醒助手。
- PDF 页面级文字批注。
- 更严格的图号字段。
- 打印记录和打印人签收。
- 数据备份和恢复。
- 企业微信或飞书通知。
- WebDAV 直连坚果云，减少对本地同步客户端依赖。
