## V3 基线

第三版已经形成可试运行的局域网签审闭环：

- 设计师网页上传 PDF，并在提交时放置设计、主管、工艺三个签名框。
- 主管和工艺固定并行审核。
- 双审通过后自动生成签后 PDF，签后文件不覆盖原审批 PDF。
- 设计师或管理员执行打印归档，签后 PDF 归档到 `05-已打印归档\项目名\`。
- 管理员可处理异常、删除图纸、批量删除、查看日志、导出 CSV、手动扫描和备份数据库。
- 系统继续以坚果云本地同步目录作为文件底座，适配 Windows 局域网运行。

截至当前，V3 后续硬化已覆盖：

- 角色入口和权限拆分。
- PDF 预览滚动、签名框跟随页面、多页签名框。
- 协同、时间线、历史版本浮窗化。
- 重新生成签后 PDF。
- 删除物理文件、历史签后文件和空项目文件夹。
- 运维诊断、服务日志、备份与追溯报表增强。

## V4 目标建议

V4 建议定位为：

> 从“可试运行的签审闭环系统”升级为“更适合长期现场使用的产品化系统”。

V4 不建议马上推翻固定双审模型，也不建议引入复杂企业级工作流。当前团队规模是 5-10 人，最有价值的是减少现场误操作、提升批量处理效率、让管理员能发现问题并恢复。

## 三种方向

### 方案 A：现场试运行加固版

重点补齐真实使用中的稳定性和可恢复性：

- 首页或管理端增加待处理风险看板。
- 签名、文件丢失、备份过期、监听异常主动提示。
- 手动扫描和备份增加更明确的结果反馈。
- 删除、作废、归档等高风险操作增加更清晰的确认和审计。
- 形成正式上线检查与日常巡检页面。

优点：风险最低，最快能提升现场可用性。

缺点：业务功能变化不大。

### 方案 B：效率与批量处理版

重点提升设计师和管理员日常操作效率：

- 多文件批量上传。
- 批量放置默认签名框模板。
- 按项目或图纸尺寸保存签名框模板。
- 每张图纸保留独立签名位置，模板只作为初始化，不能整批共享最终位置。
- 批量重新生成签后 PDF。
- 批量打印归档前检查签名状态。

优点：直接减少重复操作，适合图纸数量变多后的日常使用。

缺点：需要补充模板与批量任务的状态管理，复杂度中等。

### 方案 C：版本追溯与变更管理版

重点把图纸版本变更做成可追溯链路：

- 同一零件不同版本自动关联。
- 图纸详情显示版本树或版本列表。
- 上传新版本时提示上一版本状态。
- 驳回后重新提交可关联原审批单。
- CSV 报表和详情页增强版本历史。

优点：更贴近机械设计变更管理。

缺点：会触及版本关系模型，数据库和页面改动更大。

## 推荐路线

推荐 V4 采用“B 为主，A 为底，C 留接口”的组合。

原因：

- V3 已经解决单张图纸签审闭环，下一阶段最明显的痛点会是重复上传、重复定位签名框、重复检查签后文件。
- A 类运维加固应作为 V4 的基础，不需要做成单独大版本。
- C 类版本追溯很重要，但适合在批量上传和模板能力稳定后再正式展开，避免一次改动过大。

## V4 建议范围

### 1. 签名框模板

新增签名框模板能力：

- 管理员或设计师可把当前图纸上的三个签名框保存为模板。
- 模板包含：
  - 模板名称。
  - 适用项目名，可为空表示通用。
  - 适用图纸尺寸或页码。
  - 设计、主管、工艺三个签名框位置。
- 提交新图纸时可选择模板，一键套用三个签名框。
- 仍允许人工拖拽微调。

第一阶段只做全局模板和项目模板，不做复杂规则匹配。

### 2. 批量上传

提交图纸页支持一次选择多个 PDF：

- 每个文件独立解析文件名。
- 每个文件显示校验结果。
- 可为同一批文件设置统一项目名。
- 可为整批文件套用同一个签名框模板作为初始位置。
- 每个文件仍保留自己的签名框位置，设计师可逐张打开预览后单独调整。
- 提交时逐项校验设计、主管、工艺三个签名框是否齐全。
- 提交后逐个写入 `02-审批中\项目名\` 并创建审批单。

异常处理：

- 单个文件失败不阻塞其它文件。
- 批量提交完成后显示成功数、失败数和失败原因。
- 某张图纸缺少签名框时只影响该图纸，不影响其它签名框完整的图纸。
- 重名或版本重复时提示跳过或要求用户改名后重新提交。

### 3. 批量签后 PDF 处理

管理员和设计师在“全部图纸”中可对待打印记录做批量操作：

- 批量检查签名状态。
- 批量重新生成签后 PDF。
- 批量打开或导出签后文件清单。
- 批量标记打印归档前，必须全部满足签后 PDF 已生成。

第一阶段不直接做“一键打印”，避免依赖 Windows 打印机驱动和浏览器打印差异。

### 4. 运维风险看板

系统管理页增加 V4 风险看板：

- 审批根目录未配置。
- 标准目录缺失或不可写。
- 最近备份过期。
- 最近扫描失败。
- 签名失败待处理。
- 文件丢失待处理。
- 关键用户未配置签名。

每个风险项提供跳转入口，例如跳到异常图纸列表、用户签名状态、备份按钮或扫描按钮。

### 5. 版本追溯预留

V4 先做轻量预留，不做完整版本树：

- 在审批详情页显示同项目、同零件名的其它版本。
- 上传时如果发现同零件旧版本，提示“已有版本”。
- CSV 增加可选字段：同零件版本数、上一版本状态。

这为后续 V5 做正式版本变更管理打基础。

## 建议分批实施

### V4.1 签名框模板

交付目标：

- 能保存模板。
- 能管理模板。
- 提交图纸和详情页能套用模板。
- 保留人工微调。

验收标准：

- 从一张已定位图纸保存模板。
- 新图纸套用模板后三个签名框正确显示。
- 套用后仍可拖拽和缩放。
- 保存后签后 PDF 使用模板位置生成。

### V4.2 批量上传

交付目标：

- 多 PDF 选择、解析、校验、预览状态。
- 统一项目名。
- 批量套用签名模板作为初始位置。
- 逐张独立调整和保存签名框位置。
- 批量提交并生成多张审批单。

验收标准：

- 3 个有效 PDF 同批提交成功。
- 1 个无效 PDF 不阻塞其它文件。
- 同批不同图纸可以保存不同签名位置。
- 某一项签名框不完整时有明确提示。
- 文件名异常和重复版本有明确错误。
- 坚果云标准目录中生成对应文件。

### V4.3 批量签后处理

交付目标：

- 在全部图纸中筛选待打印图纸。
- 批量重新生成签后 PDF。
- 批量归档前检查签名状态。

验收标准：

- 多张已通过图纸批量生成签后 PDF。
- 签名缺失或签名框缺失时逐项显示失败原因。
- 批量归档只处理签后 PDF 已生成的记录。

### V4.4 运维风险看板

交付目标：

- 管理员登录后能快速看到影响上线和日常使用的风险。
- 每个风险项能跳转到处理入口。

验收标准：

- 备份过期、签名失败、文件丢失、目录不可写均能显示。
- 风险处理后看板状态刷新。
- 不影响现有系统管理页功能。

## 当前不建议纳入 V4

- CA/证书数字签名。
- 多级审批和动态审批流。
- 与坚果云 API 直接集成。
- 一键调用本地打印机自动打印。
- 复杂版本树、变更单、BOM 关联。

这些方向价值存在，但对当前 5-10 人局域网系统来说，容易把 V4 做重。

已确认核心方向：

> V4 按“签名框模板 + 批量上传”为主线推进。
