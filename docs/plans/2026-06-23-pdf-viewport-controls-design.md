# PDF viewport controls design

日期：2026-06-23

## 背景

当前批注预览和签名定位都使用自定义 PDF.js canvas 渲染，并在页面上叠加比例坐标图层。这个结构保证了批注、签名框和导出的审查版 PDF 不漂移，但预览区缺少常见的放大、缩小、适配宽度和拖动画布能力，查看机械图纸细节不方便。

## 目标

- 批注页面和签名定位页面共用一套 PDF 视图控制。
- 支持放大、缩小、100%、适配宽度、拖动浏览。
- 支持 `Ctrl + 鼠标滚轮` 缩放，普通滚轮仍用于滚动页面。
- 缩放后批注和签名框仍按比例坐标显示，不改变已保存数据。
- 不替换现有 PDF.js 渲染和业务 API。

## 推荐方案

新增轻量 `PdfViewportControls` 组件和纯函数工具，两个 workspace 复用同一套状态：

- `fit-width` 模式：保持当前按区域宽度查看的体验。
- `manual` 模式：页面宽度按 `960px * zoom` 计算，允许横向滚动。
- 拖动模式：在预览滚动容器上按住并拖动，改变 `scrollLeft` 和 `scrollTop`。

工具栏放在 PDF 区域顶部，使用图标按钮和清晰的比例显示。批注工具栏仍保留原位置，PDF 视图控制只负责看图，不改变批注工具语义。

## 边界

- 不引入 PDF.js 官方 Viewer。
- 不支持旋转、双页、页面缩略图。
- 不改变批注、签名框的数据库坐标格式。

## 验收

- 批注预览和签名定位预览都显示视图控制条。
- 点击放大/缩小后页面宽度变化，批注层和签名层跟随页面。
- 点击适配宽度后回到不横向溢出的查看方式。
- 拖动模式下可平移 PDF，不误触发批注绘制或签名框编辑。
- `npm test` 和 `npm run build` 通过。
