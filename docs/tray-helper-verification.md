# PDF 图纸审批托盘助手验收清单

该清单用于 V5 托盘助手发布前验证。

## 1. 当前机器验证边界

执行：

```powershell
.\scripts\check-tauri-prereqs.ps1
```

通过标准：

- 脚本只读执行，不创建目录、不下载依赖、不安装工具链。
- 当前机器可以验证 Node/npm、Web 前端、后端测试和托盘前端。
- 当前机器不要求补齐 Rust/MSVC，也不在本机运行 `npm run tray:build`。

外部构建机通过标准：

- Node/npm 正常。
- Rust/cargo 正常。
- Rust target 包含 `x86_64-pc-windows-msvc`。
- WebView2 Runtime 存在。
- `cl.exe` 可用。
- Visual Studio Build Tools 可被检测到。
- 可访问 npm registry 和 Rust crates 源，或配置可用的公司内部镜像。

## 2. 自动化验证

当前机器执行：

```powershell
npm test
npm run build
.\scripts\build-tray-frontend-only.ps1
```

通过标准：

- 后端和 Web 测试通过。
- 主应用生产构建通过。
- 托盘单元测试通过。
- 托盘前端构建通过。

外部构建机额外执行：

```powershell
npm install --registry=https://registry.npmmirror.com
npm --prefix apps/tray-helper install --registry=https://registry.npmmirror.com
npm test
npm run build
npm run tray:test
npm --prefix apps/tray-helper run build
npm run tray:build
```

外部构建机通过标准：

- Tauri Windows 安装包构建成功。

## 3. 服务端连通性

在用户电脑执行：

```powershell
Invoke-WebRequest http://服务器IP:8080/health
```

通过标准：

- 返回 HTTP 200。
- 响应包含 `ok: true`。

## 4. 登录流程

验证账号：

- 主管账号。
- 工艺账号。
- 设计师账号。
- 管理员账号。

通过标准：

- 正确账号密码可登录。
- 错误密码显示失败提示。
- 服务器地址不可达时显示失败提示。
- 登录成功后设置窗口隐藏。
- 重新打开托盘菜单能看到对应角色菜单。

## 5. 角色菜单

主管/工艺：

- 显示 `打开待审核`。
- 不显示 `提交图纸`。
- 不显示 `立即扫描` 和 `重启服务`。

设计师：

- 显示 `提交图纸`。
- 显示 `我的签名`。
- 不显示 `打开待审核`。
- 不显示 `立即扫描` 和 `重启服务`。

管理员：

- 显示 `打开系统管理`。
- 显示 `打开服务日志`。
- 显示 `立即扫描`。
- 显示 `重启服务`。

## 6. 通知

主管和工艺验证：

1. 准备一张待审核图纸。
2. 托盘登录主管或工艺账号。
3. 点击 `立即刷新` 或等待轮询。

通过标准：

- 出现 Windows 通知。
- 同一张图纸只提醒一次。
- 点击单张通知能打开 `#/approvals/:id`。
- 多张新任务通知能打开待办列表。
- 重启托盘助手后，已提醒的图纸不重复提醒。

设计师验证：

- 登录设计师账号。
- 确认不会收到审核提醒。

## 7. 离线与恢复

验证步骤：

1. 托盘登录成功。
2. 停止审批服务或断开访问。
3. 等待下一次轮询。
4. 恢复审批服务。

通过标准：

- 离线后托盘状态变为 `离线` 或 `异常`。
- 离线时不重复弹出错误通知。
- 恢复服务后状态回到 `在线`。

## 8. Token 过期

验证方式：

- 使用已失效 token，或修改服务端 JWT 密钥后重启服务。

通过标准：

- 托盘状态变为 `登录已过期`。
- 本地登录态被清除。
- 不继续请求认证接口。
- 重新登录后恢复轮询。

## 9. 管理员动作

管理员登录后验证：

- `打开系统管理` 能打开 `#/settings`。
- `打开服务日志` 能打开 `#/settings?tab=logs`。
- `立即扫描` 调用成功，Web 运维追溯出现扫描记录。
- `重启服务` 调用成功，Web 运维追溯出现重启请求记录。

## 10. 开机启动

当前 V5 已接入 autostart 插件，但 UI 尚未提供开关。

如果本批次需要启用开机启动，应额外验证：

- Windows 启动后托盘助手自动出现。
- 自动启动后不会弹出重复通知。
- 未登录用户启动后只显示登录设置。

## 11. 卸载

验证步骤：

1. 从 Windows 应用列表卸载托盘助手。
2. 重启电脑或注销登录。

通过标准：

- 托盘助手不再自动启动。
- 审批服务端数据不受影响。
- Web 系统仍可正常访问。

## 12. 发布结论

只有以下全部满足时，V5 托盘助手可作为发布候选：

- 自动化测试通过。
- 外部构建机产出 Tauri 安装包。
- 主管通知冒烟通过。
- 工艺通知冒烟通过。
- 设计师无审核提醒。
- 管理员菜单动作通过。
- 离线/恢复通过。
- Token 过期处理通过。
- 安装、更新、卸载流程通过。
